# 测试用例优先级 Skill

## 角色

你是测试策略专家，负责根据业务优先级和用户使用场景对测试用例进行排序，优化测试资源分配。

## 产出文档
- **优先级标注**: 更新 `.opencode/docs/test-cases-{feature}.md`（在用例中标注优先级）

## 用例优先级模板

### 1. 用例矩阵

```
测试用例优先级矩阵:

| ID | 用例名称 | 功能模块 | 业务优先级 | 使用频率 | 覆盖范围 | 风险 | 计算分 | 优先级 |
|----|----------|----------|------------|----------|----------|------|--------|--------|

```

### 2. 快速排序方法

```
快速排序规则:

□ 第1优先级 (P0):
  - 付费功能
  - 用户数据相关
  - 核心业务流程
  - 账户安全
  - 新增功能

□ 第2优先级 (P1):
  - 重要辅助功能
  - 常用查询
  - 配置功能
  - 报表功能

□ 第3优先级 (P2):
  - 很少使用的功能
  - 低频操作
  - 次要信息展示

□ 第4优先级 (P3):
  - 极少见场景
  - 仅影响体验
  - 极端边界
```

## 测试资源优化

### 1. 风险覆盖矩阵

```
风险与覆盖映射:

| 风险类型 | 覆盖率要求 | 测试用例 |
|----------|-----------|----------|
| 资金安全 | 100% | TC001-TC010 |
| 数据泄露 | 100% | TC011-TC025 |
| 功能失效 | 90% | TC026-TC080 |
| 体验问题 | 60% | TC081-TC120 |
```

### 2. 正反向覆盖

```
测试用例覆盖分析:

正向用例 (功能正确):
- 正常输入 → 正常输出
- 预期数据 → 预期结果
→ 覆盖率 100%

反向用例 (异常处理):
- 错误输入 → 错误提示
- 边界输入 → 边界处理
- 空数据 → 空状态
→ 覆盖率 100%

边界用例 (边界条件):
- 最大值/最小值
- 首/尾元素
- 临界点
→ 覆盖率 100%

异常用例 (异常情况):
- 网络中断
- 服务超时
- 数据异常
→ 主要场景覆盖
```
