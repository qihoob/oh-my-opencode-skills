# Skill: test-case-design

**角色**: 测试 (QA)
**功能**: 测试用例设计
**触发关键词**: 设计测试用例、测试场景、用例设计

## 产出文档
**路径**: `.opencode/docs/test-cases-{feature-name}.md`
**说明**: 测试用例文档，供测试执行和验收参考

## 依赖文档
**必须读取**: `.opencode/docs/requirement-{feature-name}.md`
- 从中获取验收标准（场景 1、场景 2...）
- 理解业务规则和边界条件
- 确认优先级（P0/P1/P2）

## 核心能力

1. **用例设计方法**
   - 等价类划分
   - 边界值分析
   - 场景法
   - 错误推测法

2. **用例覆盖**
   - 功能正向用例
   - 异常/边界用例
   - 兼容性用例
   - 性能相关用例

3. **用例输出**

```markdown
## 测试用例

### 用例 ID: TC001
### 用例名称: 用户登录成功
### 优先级: P0
### 前置条件: 用户已注册
### 测试步骤:
1. 输入正确用户名
2. 输入正确密码
3. 点击登录按钮
### 预期结果: 登录成功，跳转首页
### 实际结果: [填写]
### 状态: [通过/失败/阻塞]
```

## 最小上下文原则

1. **先查需求文档** - 检查 `.opencode/docs/requirement-*.md`
2. **按需读取验收标准** - 只读取与当前测试相关的场景
3. **不重复分析需求** - 直接使用已有的验收标准

## 工具可用
- read, grep, write
