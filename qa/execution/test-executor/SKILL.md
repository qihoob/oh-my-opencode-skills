# Skill: test-executor

**角色**: 测试 (QA)
**功能**: 测试执行、结果记录、Bug 自动转交
**触发关键词**: 执行测试、运行测试、测试执行、测试报告
**版本**: 2.0 (新增 Bug 自动转交机制)

## 产出文档
- **测试报告**: `.opencode/docs/test-report-{feature-name}.md`
- **Bug 单**（失败时自动生成）: `.opencode/docs/bug-{id}.md`

## 依赖文档
**必须读取**: `.opencode/docs/test-cases-{feature-name}.md`

## 核心能力

1. **测试执行**
   - 按优先级执行测试用例
   - 记录实际结果
   - 标记通过/失败状态

2. **问题记录**
   - 失败用例详细记录
   - 复现步骤
   - 环境和数据信息

3. **测试报告生成**
   - 测试覆盖率统计
   - 通过率分析
   - 风险评估

4. **Bug 自动转交（核心新增）**
   - 测试失败时自动生成结构化 Bug 单
   - Bug 单包含开发修复所需的全部上下文
   - 直接流转到 bug-coordinator 和开发

## 执行流程

### Step 1: 执行测试

```
按优先级逐条执行:
- P0 用例 → P1 用例 → P2 用例 → P3 用例
- 每条用例记录: 通过 / 失败 / 阻塞
```

### Step 2: 记录结果

```
失败的用例详细记录:
- 预期结果 vs 实际结果
- 完整复现步骤
- 环境信息（URL、版本、账号）
- 相关日志/截图
```

### Step 3: Bug 自动生成（有失败用例时触发）

```
对每条失败用例，自动生成结构化 Bug 单。

产出路径: .opencode/docs/bug-{module}-{seq}.md

Bug 单包含开发修复所需的全部信息，开发拿到即可开始分析，无需二次沟通。
```

### Step 4: 产出测试报告

### Step 5: 自动推荐下一步

```
全部通过 → 推荐 collab-acceptance-review
有失败 → 推荐 bug-coordinator（并提示已生成 Bug 单）
```
