# Skill: collab-retrospective

**角色**: 协同 (Collaboration)
**功能**: 迭代复盘
**触发关键词**: 复盘、回顾、总结、retro、迭代总结

## 核心能力

1. **数据收集**
   - 迭代计划 vs 实际完成
   - Bug 统计与趋势
   - 代码审查问题统计

2. **问题识别**
   - 做得好的 (Keep)
   - 需要改进的 (Improve)
   - 停止做的 (Stop)

3. **改进计划**
   - 制定具体改进行动
   - 分配负责人
   - 设定时间节点

## 产出文档
- **复盘报告**: `.opencode/docs/retrospective-report.md`

## 输出格式

```markdown
# 迭代复盘报告

## 基本信息
- **迭代周期**: Sprint [N] ([开始日期] - [结束日期])
- **参与人员**: [产品、开发、测试]
- **会议时间**: [日期]

## 迭代目标回顾

### 计划完成
| 需求 | 优先级 | 状态 | 备注 |
|------|--------|------|------|
| 用户登录 | P0 | ✅ 完成 | |
| 第三方登录 | P1 | ⚠️ 部分完成 | 微信登录延期 |
| 登录日志 | P2 | ❌ 未完成 | 资源不足 |

### 实际完成
- **完成率**: 66% (2/3 需求完成)
- **故事点**: 21/30

## 质量数据

### Bug 统计
- **新增 Bug**: 15 个
- **已修复**: 12 个
- **遗留**: 3 个
- **Bug 密度**: 0.5 Bug/故事点

### 代码审查
- **审查次数**: 8 次
```
