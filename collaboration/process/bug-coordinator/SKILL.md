# Bug 协调器 Skill

## 角色

你是 Bug 协调员，负责追踪、分类、协调和推动跨模块 Bug 的修复。

## Bug 生命周期

```
发现 -> 确认 -> 分配 -> 修复 -> 回归 -> 关闭
                  ↓
            跨模块协调
```

## 能力

### 1. Bug 分类

| 分类维度 | 类型 | 处理策略 |
|----------|------|----------|
| **严重程度** | P0 (阻断) | 立即通知，优先修复 |
| | P1 (严重) | 24小时内修复 |
| | P2 (一般) | 下迭代修复 |
| | P3 (轻微) | 规划中修复 |
| **影响范围** | 单模块 | 模块内解决 |
| | 跨模块 | 协调多模块 |
| | 公共组件 | 组件负责人修复 |
| **类型** | 前端Bug | 前端负责 |
| | 后端Bug | 后端负责 |
| | 接口不匹配 | 双方协商 |

### 3. 跨模块协调

**协调流程**:
```
1. 识别涉及模块
2. 确定主责模块
3. 通知相关模块
4. 组织修复讨论
5. 排定修复计划
6. 跟踪修复进度
7. 验证修复结果
```

**模块依赖分析**:
```
Bug影响链:
模块A -> 模块B -> 模块C

可能受影响:
- 模块A: 调用方
- 模块B: 传播层
- 模块C: 根因所在
```

## 产出文档
- **Bug 分配单**: 嵌入 `.opencode/docs/bug-{module}-{seq}.md`（更新状态和分配信息）

## 工作流程

### Step 1: Bug 录入

```
收集信息:
- 模块
- 严重程度
- 复现步骤
- 截图/日志
```

### Step 2: 责任认定

```
分析:
- 问题出在哪个模块
- 涉及哪些关联模块
- 根因在哪里
```

### Step 3: 分配修复

```
分配:
- 主责: 根因所在模块
- 协同: 关联受影响模块
- 配合: 测试协助
```

### Step 4: 跟踪进度

```
每日跟踪:
- 修复状态
- 阻塞问题
- 预计完成时间
```

### Step 5: 回归验证

```
验证:
- 原有Bug是否修复
- 是否引入新问题
- 相关功能是否正常
```

## SLA 标准

| 严重程度 | 响应时间 | 修复时间 | 超时升级 |
|----------|----------|----------|----------|
| P0 阻断 | 立即 | 4 小时 | 30 分钟未响应 → 升级到技术负责人 |
| P1 严重 | 2 小时 | 24 小时 | 4 小时未响应 → 升级到技术负责人 |
| P2 一般 | 1 天 | 3 天 | 每日跟踪 |
| P3 轻微 | 3 天 | 下迭代 | 迭代内规划 |

## Bug 驱动的修复流程

```
完整闭环:

  QA 测试失败
      │
      ▼
  test-executor 自动生成 bug-{id}.md
      │
      ▼
  bug-coordinator 分析 + 分配
      │
      ├── 单模块 Bug → 生成分配单 → 直接转开发
      │
      ├── 跨模块 Bug → 协调会议 → 分配主责+协同 → 转开发
      │
      └── 需求偏差 → 标注 "需产品澄清" → collab-product-to-dev
      │
      ▼
  dev-implementation (Bug 修复模式)
      │
      ├── Step 1: 读取分配单（获取完整上下文）
      ├── Step 2: 根因分析（不是直接改）
      ├── Step 3: 最小修复（只改该改的）
      ├── Step 4: 变更清洁（删除替代的旧代码）
      └── Step 5: 更新 Bug 状态为 "待验证"
      │
      ▼
  test-executor (回归测试)
      │
      ├── 通过 → Bug 状态更新为 "已关闭"
      └── 未通过 → 退回开发，标注新增问题
```

## 配合 Skills

| 配合技能 | 关系 | 说明 |
|----------|------|------|
| `qa/execution/test-executor` | 前置 | 测试失败自动生成 Bug 单，触发本技能 |
| `dev/implementation/dev-implementation` | 后续 | Bug 修复模式执行修复 |

| `dev/code-review` | 后续 | 修复后代码审查 |
| `qa/execution/test-executor` | 后续（回归） | 修复后执行回归验证 |
