# Skill: collab-acceptance-review

**角色**: 协同 (Collaboration)
**功能**: 验收评审（含项目级健康总评）
**触发关键词**: 验收、UAT、验收评审、产品验收、上线确认
**版本**: 2.0 (新增项目级检查点)

## 核心能力

1. **验收准备**
   - 确认需求文档验收标准
   - 准备验收测试用例
   - 组织验收会议

2. **验收执行**
   - 逐项验证验收标准
   - 记录验收问题
   - 评估问题严重程度

3. **验收决策**
   - 通过/有条件通过/不通过
   - 制定问题修复计划
   - 确认上线时间

## 输出格式

```markdown
# 验收评审报告

## 基本信息
- **功能名称**: [名称]
- **验收时间**: [日期]
- **验收人员**: [产品、开发、测试]
- **验收环境**: [UAT 环境]

## 验收结果概览
| 验收项 | 总数 | 通过 | 失败 | 通过率 |
|--------|------|------|------|--------|
| 验收标准 | 10 | 8 | 2 | 80% |

## 详细验收情况

### ✅ 通过的验收项
1. AC1: 用户可以正常登录
2. AC2: 密码错误有明确提示
3. AC3: 登录状态持久化

### ❌ 未通过的验收项
1. AC5: 第三方登录
   - **问题描述**: 微信登录回调失败
   - **严重程度**: 中
   - **修复建议**: 检查回调 URL 配置
   - **负责人**: @开发
   - **预计修复**: [日期]

2. AC7: 登录日志记录
   - **问题描述**: 日志未记录登录 IP
   - **严重程度**: 低
   - **修复建议**: 补充 IP 字段
   - **负责人**: @开发
   - **预计修复**: [日期]

## 项目级健康总评（验收前快照）

| 维度 | 状态 | 说明 |
|------|------|------|
| 需求覆盖 | 🟢 全部AC已验证 | 10/10 |
| 文档一致性 | 🟡 | test-report 2天前未更新 |
| 契约一致性 | 🟢 | contract-*.md 与实现对齐 |
| 变更清洁 | 🟢 | 无残留旧代码 |
| 跨层对齐 | 🟡 | 前端1处错误码未处理 |

### 健康问题（需在上线前解决）
- [ ] {问题} → 负责人: {name}

## 验收结论
- [ ] ✅ **通过** - 可上线
- [ ] ⚠️ **有条件通过** - 修复问题后可上线，无需复测
- [ ] 🔄 **需要复测** - 修复问题后需重新验收
- [ ] ❌ **不通过** - 存在严重问题，延期上线

## 上线决策
- **建议上线时间**: [日期]
- **上线版本**: [v1.0.0]
- **回滚方案**: [说明]

## 后续行动
- [ ] 开发修复 AC5 问题 ([日期])
- [ ] 开发修复 AC7 问题 ([日期])
- [ ] 测试验证修复 ([日期])
- [ ] 产品确认上线 ([日期])
- [ ] 执行 global-project-analysis 健康检查 (上线前最终确认)
```
