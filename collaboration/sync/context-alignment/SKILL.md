# 同步对齐 Skill

**角色**: 协同 (Collaboration)
**功能**: 上下文对齐、需求对齐、模块协调
**触发关键词**: 同步、对齐、上下文对齐、确认范围、模块协调

## 核心能力

### 1. 上下文对齐
- 背景同步
- 范围对齐
- 理解一致

### 2. 需求对齐
- 需求澄清
- 优先级确认
- 变更通知

### 3. 模块协调
- 依赖协调
- 接口对齐
- 进度同步

## 执行步骤

### Step 1: 收集各方状态

```
对每个参与方：
- 当前进度
- 遇到的问题
- 待确认事项
- 依赖等待
```

### Step 2: 识别对齐点

```
检查：
- 各方理解是否一致？
- 范围是否有偏移？
- 依赖是否阻塞？
- 优先级是否对齐？
```

### Step 3: 达成共识

```
对每个对齐点：
- 澄清分歧
- 确认决议
- 分配行动项
- 约定下次同步时间
```

## 输出格式

```markdown
## 同步记录

### 主题: {同步主题}

### 参与方
- {角色1}: {name}
- {角色2}: {name}

### 同步内容
| 事项 | 状态 | 备注 |
|------|------|------|
| {事项1} | {状态} | {备注} |
| {事项2} | {状态} | {备注} |

### 依赖问题
- 等待: {阻塞项}
- 风险: {风险描述}

### 决议
1. {决议1}
2. {决议2}

### 行动项
| 行动 | 负责人 | 截止时间 |
|------|--------|----------|
| {行动1} | {name} | {date} |
```
