# Skill: collab-dev-to-qa

**角色**: 协同 (Collaboration)
**功能**: 开发到测试的提测交接
**触发关键词**: 提测、测试交接、开发完成、申请测试

## 核心能力

1. **提测申请**
   - 说明功能完成情况
   - 提供测试账号/数据
   - 标注风险点

2. **测试范围确认**
   - 影响模块说明
   - 回归测试建议
   - 特殊场景提示

3. **环境准备**
   - 测试环境确认
   - 数据准备说明
   - 部署版本标识

## 输出格式

```markdown
## 提测申请单

### 基本信息
- **功能名称**: [名称]
- **开发完成时间**: [日期]
- **提测版本**: [v1.0.0]
- **开发负责人**: [姓名]

### 完成情况
- [ ] 功能开发完成
- [ ] 单元测试通过
- [ ] 代码审查完成
- [ ] 自测通过

### 测试建议
#### 重点测试范围
- 功能点 1
- 功能点 2

#### 回归测试建议
- 影响模块 A
- 影响模块 B

#### 特殊场景
- 场景说明 1
- 场景说明 2

### 测试数据
- **测试账号**: test_user / password123
- **测试环境**: http://test-env.com
- **数据准备**: 预置订单数据

### 风险说明
- 风险点 1（已规避/待验证）
- 风险点 2（已规避/待验证）

### 预计测试周期
[X] 个工作日
```

## 工具可用
- read, write, grep
