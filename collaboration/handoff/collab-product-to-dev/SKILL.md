# Skill: collab-product-to-dev

**角色**: 协同 (Collaboration)
**功能**: 产品到开发的需求交接
**触发关键词**: 需求交接、需求评审、开发确认、转开发、交给开发
**版本**: 2.0 (增强协作)

## 依赖文档
- **必须读取**: `.opencode/docs/requirement-{feature}.md`


## 产出文档
- **需求交接单**: `.opencode/docs/handoff-product-to-dev.md`

## 核心能力

1. **需求澄清**
   - 详细说明功能需求
   - 解答开发疑问
   - 提供参考示例
   - 验收标准确认

2. **技术对齐**
   - 确认技术实现方案
   - 识别技术风险
   - 讨论替代方案
   - 评估工作量

3. **交接确认**
   - 明确完成标准
   - 约定后续沟通机制
   - 记录决策要点
   - 生成交接单

## 前置依赖

**输入文档**:
- `.opencode/docs/requirement-*.md` (需求文档)
- 或：`.opencode/docs/product-technical-assessment-*.md` (技术评估)

**输出文档**:
- `.opencode/docs/handoff-product-to-dev.md` (交接单)

## 协作流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  需求文档   │ →  │  需求澄清   │ →  │  技术对齐   │
│  (已有)     │    │  (问答)     │    │  (方案确认) │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                              │
                                              ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  后续技能   │ ←  │  交接单生成  │ ←  │  双方确认   │
│ dev-context │    │  (输出)     │    │  (签字)     │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 交接检查清单

### 需求完整性
- [ ] 用户故事清晰
- [ ] 验收标准明确
- [ ] 边界条件说明
- [ ] 异常流程覆盖

### 技术可行性
- [ ] 技术方案确认
- [ ] 依赖模块识别
- [ ] 风险评估完成
- [ ] 工作量估算合理

### 交付标准
- [ ] 完成定义 (DoD) 明确
- [ ] 测试要求说明
- [ ] 性能指标定义
- [ ] 监控要求说明
