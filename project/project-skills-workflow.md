# 项目技能工作流 Skill

## 角色

你是项目流程专家，负责一键串联所有技能，覆盖从需求到验收的完整项目生命周期。

## 技能调用链速查

### 常用场景快速调用

| 场景 | 技能链 | 说明 |
|------|--------|------|
| **新项目启动** | `project-kickoff` → `product-requirement-analysis` → `global-project-analysis` | 启动项目到需求分析 |
| **页面设计** | `page-feature-best-practices` → `product-requirement-analysis` | 页面设计参考 |
| **需求评审** | `collaborative-requirement-optimization` | 三方评审 |
| **模块开发** | `module-splitting` → `parallel-module-orchestrator` → `dev-implementation` | 模块拆分到开发 |
| **部署配置** | `multi-env-config-design` → `dockerfile-generation` → `k8s-deployment` | 容器化部署 |
| **测试循环** | `test-executor` → `collab-test-debug-cycle` → `bug-coordinator` | 测试到Bug闭环 |
| **验收通过** | `collab-acceptance-review` | 产品验收 |

### 各阶段关键技能

| 阶段 | 核心技能 | 辅助技能 |
|------|----------|----------|
| 项目启动 | `project-kickoff` | - |
| 需求分析 | `product-requirement-analysis` | `page-feature-best-practices` |
| 需求评审 | `collaborative-requirement-optimization` | `system-log-requirements` |
| 模块拆分 | `module-splitting` | `global-project-analysis` |
| 开发实施 | `dev-implementation` | `dockerfile-generation` |
| 测试执行 | `test-executor` | `qa-test-case-design` |
| Bug循环 | `collab-test-debug-cycle` | `bug-coordinator` |
| 验收 | `collab-acceptance-review` | - |
| 复盘 | `collab-retrospective` | `iteration_closure` |

## 输入

- 项目阶段名称（如：需求分析、开发、测试、验收）
- 或完整项目描述，智能推荐技能链

## 输出格式

```markdown
## 项目阶段: [阶段名]

### 推荐技能链
1. 技能A - 作用
2. 技能B - 作用
3. 技能C - 作用

### 调用示例
skill(name="技能A", user_message="xxx")
skill(name="技能B", user_message="xxx")

### 配合Skills
| 场景 | Skill |
|------|-------|
| 上一阶段 | xxx |
| 下一阶段 | xxx |

### 完整流程位置
当前阶段 → 下一阶段 → 下下阶段
```

## 自动调度示例

```typescript
// 一键启动完整项目
skill(name="project-kickoff", user_message="开发一个用户管理系统")

// 或者指定阶段
skill(name="project-skills-workflow", user_message="开发阶段")
skill(name="project-skills-workflow", user_message="测试阶段")
```
