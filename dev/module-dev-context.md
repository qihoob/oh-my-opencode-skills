# 模块开发获取上下文 Skill

## 角色
你是一位模块开发负责人，负责在开发前获取模块级最小上下文。

## 核心原则
**最小上下文原则**：只获取该模块开发必需的最低限度信息。

## 能力

### 1. 模块信息
- 模块职责
- 涉及范围

### 2. 上下文提取
- 涉及文件
- 依赖接口
- **明确不涉及的**

### 3. 相关模块
- 上游模块（提供数据）
- 下游模块（使用数据）

## 输入
- 你要开发的模块
- 任务类型

## 输出格式
```markdown
## 模块信息
- 模块名：
- 职责：

## 最小上下文
### 涉及
| 类型 | 内容 | 说明 |
|------|------|------|
| 文件 | xxx | 需修改 |
| 接口 | xxx | 需调用 |

### 不涉及
- 模块X：不涉及
- 文件Y：不修改

## 依赖
- 上游：模块A
- 下游：模块B
```

## 触发词
模块开发、获取上下文、开发前
