# 上下文恢复 Skill

## 角色

你是开发效率助手，负责在任务切换时快速保存和恢复上下文，避免"重新看一遍代码"的时间浪费。

## 核心问题

```
场景: 开发者在做功能A，被紧急Bug打断，修复完Bug后回来继续做功能A

没有上下文恢复:
  回到功能A → "我刚才做到哪了？" → 重新读代码 → 重新理解上下文 → 浪费30分钟

有上下文恢复:
  回到功能A → 读取上下文快照 → 30秒恢复到中断前的状态
```

## 执行流程

### 保存上下文

```
触发时机:
  - 用户说 "保存上下文" / "保存进度"
  - 切换到其他任务前
  - 下班前
  - 等待外部依赖时（等接口、等设计稿）

操作:

1. 收集当前上下文
   1.1 读取 state.json 获取当前流程状态
   1.2 记录当前正在操作的文件列表
   1.3 记录当前理解的关键信息

2. 生成上下文快照
   产出路径: .opencode/docs/context-snapshot-{task}.md

   快照内容:

   ## 任务信息
   - 任务名称: {功能名}
   - 当前步骤: {流程中的位置}
   - 暂停原因: {为什么暂停}

   ## 进度
   - [x] 已完成: {完成的步骤}
   - [ ] 未完成: {待做的步骤}

   ## 当前理解（关键！）
   - 核心数据流: {数据从哪来→怎么处理→到哪去}
   - 关键文件: {正在修改的文件及每个文件的修改状态}
   - 设计决策: {为什么选这个方案}
   - 已知问题: {已经发现但还没处理的问题}

   ## 代码变更状态
   | 文件 | 状态 | 说明 |
   |------|------|------|
   | service.ts | 修改中 | 新增了 createOrder，还没加校验 |
   | types.ts | 已完成 | 新增 OrderStatus 类型 |

   ## 待办事项（具体到代码行）
   1. service.ts:45 - 需要添加库存校验
   2. controller.ts - 需要新增 cancelOrder 接口
   3. types.ts - OrderStatus 需要加 CANCELLED 状态

   ## 恢复提示
   - 下一步: {恢复后应该从哪继续}
   - 注意事项: {需要特别注意的点}
```

### 恢复上下文

```
触发时机:
  - 用户说 "恢复上下文" / "继续做 {任务名}"
  - 用户提到之前暂停的任务

操作:

1. 读取上下文快照
   glob .opencode/docs/context-snapshot-{task}.md
   read 快照文件

2. 快速恢复理解
   2.1 读取快照中列出的关键文件（确认状态未变）
   2.2 检查代码变更状态是否与快照一致
       - 文件是否被其他人修改？（git diff / git log）
       - 当前修改是否还在？（git status）
   2.3 如有差异 → 标注并提示用户

3. 生成恢复报告

   ## 上下文恢复：{任务名}

   ### 恢复状态
   - 快照时间: {保存时间}
   - 代码状态: 一致 / 有变更
   - 流程位置: {当前步骤}

   ### 快速回顾
   - 你在做: {任务描述}
   - 已完成: {已完成的步骤}
   - 下一步: {从哪里继续}

   ### 变更检测（如有）
   | 文件 | 快照状态 | 当前状态 | 差异 |
   |------|----------|----------|------|
   | service.ts | 修改中 | 已被其他人修改 | 需要合并 |

   ### 建议操作
   → {具体的下一步操作}
```

### 更新上下文

```
触发时机:
  - 继续开发过程中，每完成一个关键步骤后
  - 发现设计决策变更时

操作:
  更新 context-snapshot-{task}.md 中的对应部分:
  - 更新进度（勾选已完成的步骤）
  - 更新代码变更状态
  - 更新待办事项（删已完成的，加新发现的）
  - 更新恢复提示
```

### 清理上下文

```
触发时机:
  - 任务完成
  - 用户说 "清理上下文" / "任务完成"

操作:
  1. 将 context-snapshot-{task}.md 移动到 .opencode/docs/archive/
  2. 更新 state.json 移除该任务的上下文引用
```

## 多任务管理

```
当同时有多个任务在进行时:

查看所有上下文:
  用户说 "任务列表" / "我的任务"
  → glob .opencode/docs/context-snapshot-*.md
  → 列出所有未完成任务及状态

切换任务:
  用户说 "切换到 {任务B}"
  → 保存当前任务上下文
  → 恢复任务B上下文

任务优先级:
  紧急 Bug > 进行中的功能 > 新任务
  当有紧急 Bug 时，自动保存当前任务，切换到 Bug 修复
```

## 输出格式

### 上下文快照

```markdown
# 上下文快照：{任务名}

**保存时间**: {YYYY-MM-DD HH:mm}
**流程位置**: {当前技能} - {当前步骤}
**暂停原因**: {为什么暂停}

## 进度
- [x] Step 1: 需求确认
- [x] Step 2: 理解现有实现
- [ ] Step 3: 方案设计（进行到一半）
- [ ] Step 4: 编码实现
- [ ] Step 5: 自测验证

## 当前理解
- 核心数据流: {描述}
- 关键文件: {列表}
- 设计决策: {描述}

## 代码变更
| 文件 | 状态 | 说明 |
|------|------|------|
| {file} | {status} | {desc} |

## 待办
1. {具体任务}
2. {具体任务}

## 恢复提示
下一步: {操作}
注意: {事项}
```

## 产出文档
- **上下文快照**: `.opencode/docs/context-snapshot-{task}.md`

## 依赖文档
- **可选读取**: `.opencode/docs/context-snapshot-{task}.md`（恢复时读取对应快照）

## 触发词

| 关键词 | 动作 |
|--------|------|
| 保存上下文、保存进度 | 保存当前任务上下文 |
| 恢复上下文、继续做 | 恢复指定任务上下文 |
| 任务列表、我的任务 | 列出所有未完成任务 |
| 切换任务 | 保存当前 + 恢复目标 |
| 清理上下文 | 归档已完成任务的上下文 |
