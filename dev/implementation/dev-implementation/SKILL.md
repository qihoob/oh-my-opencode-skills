# Skill: dev-implementation

**角色**: 开发 (Dev)
**功能**: 功能实现
**触发关键词**: 实现功能、编写代码、开发功能

## 产出文档
**路径**: `.opencode/docs/implementation-{feature-name}.md`
**说明**: 实现完成后记录技术细节，供测试/验收参考

## 依赖文档
**必须读取**: `.opencode/docs/requirement-{feature-name}.md`
- 从中获取验收标准
- 理解功能边界条件
- 确认优先级

## 核心能力

1. **需求理解**
   - 从需求文档获取验收标准
   - 识别边界条件
   - 确认优先级

2. **代码实现**
   - 遵循项目规范
   - 合理的代码结构
   - 必要的注释

3. **自测验证**
   - 基本功能验证
   - 边界条件检查
   - 错误处理验证

## 开发流程

1. **读取需求文档** → 2. 获取代码上下文 → 3. 设计实现方案 → 4. 编写代码 → 5. 自测验证 → 6. 写入实现文档

## 最小上下文原则

1. **先查文档** - 检查 `.opencode/docs/` 是否有相关需求文档
2. **按需加载** - 只读取与当前功能相关的文档章节
3. **不重复获取** - 已有的需求文档不再重新分析

## 工具可用
- read, write, edit, glob, grep, lsp_diagnostics
