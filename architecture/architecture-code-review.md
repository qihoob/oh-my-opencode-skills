## 能力

### 1. 架构合理性评审

```markdown
## 架构评审检查点

### 分层架构
- [ ] Controller层: 职责单一，只做请求处理
- [ ] Service层: 业务逻辑完整，无重复代码
- [ ] Repository/DAO层: 数据访问抽象合理
- [ ] Model层: 实体定义清晰，关系正确

### 设计模式
- [ ] 合理使用设计模式（如工厂、策略、装饰器）
- [ ] 避免过度设计
- [ ] 单一职责原则遵守

### 模块划分
- [ ] 模块边界清晰
- [ ] 依赖关系合理，无循环依赖
- [ ] 公共代码提取适当
```

### 2. 代码质量评审

```markdown
## 代码质量检查

### 可读性
- [ ] 命名语义清晰 (变量/函数/类名)
- [ ] 函数长度合理 (<80行)
- [ ] 注释恰当 (为什么，而非做什么)
- [ ] 复杂逻辑有说明

### 简洁性
- [ ] 避免重复代码 (DRY原则)
- [ ] 避免过长参数列表
- [ ] 避免深层嵌套 (>3层)
- [ ] 简化条件表达式

### 错误处理
- [ ] 异常捕获合理
- [ ] 错误信息明确
- [ ] 资源正确释放
```

### 3. 性能与安全评审

```markdown
## 性能安全检查

### 性能
- [ ] 数据库查询优化 (避免N+1)
- [ ] 合理使用缓存
- [ ] 大数据量处理 (分页/流式)
- [ ] 避免同步阻塞

### 安全

### 5. 代码示例评审

```markdown
## 示例评审

### 原始代码
```java
public User getUser(Long id) {
    User user = userMapper.selectById(id);
    if (user != null) {
        user.setRoles(roleMapper.selectByUserId(id));
    }
    return user;
}
```

### 问题分析
❌ N+1查询问题
❌ 缺乏异常处理

### 优化建议
```java
public User getUser(Long id) {
    User user = userMapper.selectById(id);
    if (user == null) {
        throw new UserNotFoundException(id);
    }
    // 使用JOIN一次性查询
    user.setRoles(roleMapper.selectByUserId(id));
    return user;
}
```
```
