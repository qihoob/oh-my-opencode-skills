# Skill: document-integrity-check

**角色**: 系统 (System) / 质量保证
**功能**: 文档完整性与一致性检查
**触发关键词**: 文档检查、完整性检查、文档审计、文档质量

## 产出文档
**路径**: `.opencode/docs/DOC_AUDIT.md`
**说明**: 文档审计报告，记录完整性和一致性问题的详细分析

## 依赖文档
**扫描范围**: `.opencode/docs/` 全部内容

## 核心能力

1. **全局扫描**
   - 扫描所有需求文档
   - 扫描所有实现文档
   - 扫描所有测试文档
   - 建立文档关系图

2. **完整性检查清单**
   ```
   □ 每个需求是否有验收标准？
   □ 每个功能是否有实现文档？
   □ 每个验收标准是否有测试用例？
   □ 文档是否有最后更新时间？
   □ 文档是否有明确的负责人？
   ```

3. **一致性检查**
   - 需求变更是否有对应实现更新？
   - 实现修改是否有对应测试更新？
   - 测试失败是否有对应问题记录？

4. **质量评分**
   - 文档覆盖率 = 有文档的功能数 / 总功能数
   - 测试覆盖率 = 有测试的功能数 / 总功能数
   - 一致性得分 = 一致的文档对数 / 总文档对数

## 输出格式

```markdown
# 文档审计报告

**审计时间**: 2026-03-09
**审计范围**: .opencode/docs/

## 质量评分
| 指标 | 得分 | 说明 |
|------|------|------|
| 文档覆盖率 | 85% | 17/20 功能有文档 |
| 测试覆盖率 | 70% | 14/20 功能有测试 |
| 一致性得分 | 90% | 27/30 文档对一致 |

## 问题清单

### 🔴 严重问题
1. requirement-login.md 缺少验收标准
2. implementation-payment.md 实现与需求不符

### 🟡 警告
1. test-cases-user.md 测试用例过期（最后更新 30 天前）
2. requirement-order.md 需求变更但实现文档未更新

### 🟢 建议
1. 建议添加文档最后审查日期
2. 建议统一文档命名规范

## 缺失文档列表
| 功能 | 缺失文档 | 影响 |
|------|----------|------|
| 支付 | implementation-payment.md | 无法验证实现 |
| 退款 | test-cases-refund.md | 无测试覆盖 |
| 通知 | requirement-notify.md | 需求未文档化 |

## 行动建议
1. 优先补充支付模块实现文档
2. 更新过期的测试用例
3. 审查需求变更同步流程
```

## 检查规则

### 需求文档规则
- 必须包含用户故事
- 必须包含验收标准（至少 1 个场景）
- 必须包含优先级

### 实现文档规则
- 必须引用对应的需求文档
- 必须说明技术实现方案
- 必须说明已知限制

### 测试文档规则
- 必须引用对应的需求文档
- 必须覆盖所有验收标准
- 必须包含测试步骤和预期结果

## 工具可用
- glob: 扫描文档目录
- read: 读取文档内容
- write: 写入审计报告
- grep: 搜索关键词
