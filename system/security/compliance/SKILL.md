# Skill: security-compliance

**角色**: 系统 (System) / 技术负责人
**功能**: 安全合规检查
**触发关键词**: 安全审计、合规检查、GDPR、数据加密、SOC2、安全漏洞

## 产出文档
**路径**: `.opencode/docs/security-audit-{date}.md`
**说明**: 安全审计报告，包含漏洞扫描结果、合规检查清单、修复建议

## 依赖文档
- 项目源代码
- **必须读取**: `.opencode/docs/project-overview.md`（项目概览）`

## 核心能力

1. **漏洞扫描**
   - SQL 注入检测
   - XSS 跨站脚本检测
   - CSRF 跨站请求伪造检测
   - 敏感信息泄露检测
   - 认证授权漏洞检测

2. **合规检查**
   - GDPR 数据保护合规
   - SOC2 安全控制合规
   - PCI-DSS 支付卡行业合规
   - 数据加密合规

3. **代码审计**
   - 输入验证检查
   - 输出编码检查
   - 密码学使用检查
   - 日志安全审计

## 检查清单

### 应用安全
- [ ] 所有用户输入都经过验证和过滤
- [ ] 所有输出都经过编码处理
- [ ] 使用参数化查询防止 SQL 注入
- [ ] 实现 CSRF 令牌保护
- [ ] 会话管理安全（超时、重置）

### 数据安全
- [ ] 敏感数据加密存储（密码、密钥、PII）
- [ ] 传输层加密（HTTPS/TLS）
- [ ] 密钥安全管理（不使用硬编码密钥）
- [ ] 数据脱敏处理

### 认证授权
- [ ] 密码强度策略
- [ ] 多因素认证支持
- [ ] 最小权限原则
- [ ] 访问控制列表

### 合规要求 (GDPR)
- [ ] 用户同意管理
- [ ] 数据访问权利实现
- [ ] 被遗忘权实现
- [ ] 数据可携带性
- [ ] 隐私政策文档

### 合规要求 (SOC2)
- [ ] 访问控制审计日志
- [ ] 变更管理流程
- [ ] 风险评估文档
- [ ] 业务连续性计划

## 严重问题

### SQL-001: SQL 注入风险
- **位置**: `src/user/controller.ts:45`
- **描述**: 用户输入直接拼接到 SQL 查询
- **修复建议**: 使用参数化查询
- **代码示例**:
  ```typescript
  // ❌ 错误
  const query = `SELECT * FROM users WHERE id = ${userId}`;

  // ✅ 正确
  const query = 'SELECT * FROM users WHERE id = ?';
  db.execute(query, [userId]);
  ```
