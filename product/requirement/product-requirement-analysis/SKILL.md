# Skill: product-requirement-analysis

**角色**: 产品 (Product)
**功能**: 需求分析与用户故事编写
**触发关键词**: 需求分析、用户故事、需求澄清、功能定义

## 产出文档
**路径**: `.opencode/docs/requirement-{feature-name}.md`
**说明**: 需求分析完成后必须写入此文件，供后续开发/测试技能读取

## 依赖文档
- 无（首次分析）
- 或：`.opencode/docs/project-overview.md`（如有项目概览）

## 核心能力

1. **需求理解**
   - 将模糊需求转化为清晰的用户故事
   - 识别核心用户价值和业务目标
   - 区分 MVP 和扩展功能

2. **验收标准制定**
   - 定义可测试的验收标准 (AC)
   - 识别边界条件和异常流程
   - 编写 Gherkin 风格场景

3. **需求输出**
   - 用户故事模板：As a... I want to... So that...
   - 清晰的功能描述和限制条件
   - 优先级建议 (MoSCoW)

## 输入要求

用户需提供：
- 原始需求描述
- 目标用户画像（可选）
- 业务背景（可选）

## 输出格式

```markdown
# 需求文档：{功能名称}

## 用户故事
As a [用户角色], I want to [功能], so that [价值]

## 验收标准
### 场景 1: [正常流程]
- Given [前置条件]
- When [触发动作]
- Then [预期结果]

### 场景 2: [异常处理]
- Given [异常条件]
- When [触发动作]
- Then [预期结果/错误提示]

## 优先级
- Must Have: [...]
- Should Have: [...]
- Could Have: [...]
- Won't Have: [...]

## 后续技能引用
<!-- 开发技能请读取此文档获取验收标准 -->
<!-- 测试技能请读取此文档设计测试用例 -->
```

## 最小上下文原则

1. **不加载无关文件** - 只读取与当前需求相关的现有文档
2. **输出结构化** - 便于后续技能按需读取特定章节
3. **文档引用标记** - 在输出中标注哪些内容供哪个技能使用

## 工具可用
- file-read: 读取现有需求文档
- grep: 搜索相关代码/文档
- file-write: 写入需求文档到 `.opencode/docs/`
