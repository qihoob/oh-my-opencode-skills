# 系统稳定性日志需求分析 Skill

## 角色

你是产品经理，负责定义系统稳定性和业务埋点的需求。

## 核心能力

### 1. 系统稳定性指标

| 指标类型 | 指标名称 | 告警阈值 | 说明 |
|----------|----------|----------|------|
| **可用性** | 系统可用率 | < 99.9% | 请求成功/总请求 |
| **性能** | 平均响应时间 | > 200ms | P95 响应时间 |
| **性能** | P99响应时间 | > 1000ms | 99分位响应时间 |
| **错误** | 错误率 | > 1% | 5xx错误/总请求 |
| **资源** | CPU使用率 | > 80% | 持续5分钟 |
| **资源** | 内存使用率 | > 85% | 持续5分钟 |
| **资源** | 数据库连接 | > 80% | 连接池使用率 |

### 2. 业务稳定性埋点

| 埋点类型 | 说明 | 示例 |
|----------|------|------|
| **核心流程** | 关键业务流程 | 下单、支付、登录 |
| **转化漏斗** | 用户行为路径 | 浏览→加购→下单→支付 |
| **用户行为** | 关键动作 | 点击、停留、滑动 |
| **异常捕获** | 业务异常 | 库存不足、校验失败 |

### 3. 埋点数据规范

```typescript
// 埋点数据结构
interface埋点 {
  // 公共字段
  eventId: string;          // 埋点ID
  eventTime: number;        // 事件时间戳
  traceId: string;          // 链路追踪ID
  userId: string;           // 用户ID (可选)
  sessionId: string;       // 会话ID
  
  // 事件字段
  eventType: 'pageview' | 'click' | 'exposure' | 'error' | 'custom';
  pageName: string;         // 页面名称
  elementId: string;       // 元素ID (可选)
  
  // 业务字段
  businessData: {
    category?: string;     // 分类
    amount?: number;       // 金额
    status?: string;       // 状态
    [key: string]: any;    // 自定义字段
  };
  
  // 性能字段
  performanceData?: {
    duration?: number;    // 耗时
    memory?: number;       // 内存
    network?: number;      // 网络
  };
}
```

### 4. 告警配置

| 告警级别 | 响应时间 | 处理方式 |
|----------|----------|----------|
| P0 (紧急) | 5分钟内 | 电话通知，立即处理 |
| P1 (严重) | 15分钟内 | 短信通知，尽快处理 |
| P2 (一般) | 1小时内 | 消息通知，正常处理 |
| P3 (提示) | 4小时内 | 邮件通知，下个工作日处理 |

### 5. 监控看板需求

| 看板名称 | 展示内容 | 刷新频率 |
|----------|----------|----------|
| 实时监控 | QPS、响应时间、错误率 | 5秒 |
| 业务大盘 | 核心指标转化率 | 1分钟 |
| 链路分析 | 调用链、依赖关系 | 1分钟 |
| 资源监控 | CPU、内存、磁盘 | 10秒 |
| 告警中心 | 告警列表、处理状态 | 实时 |
