# Skill: module-product-requirement

**角色**: 产品 (Product)
**功能**: 模块需求细化
**触发关键词**: 模块需求、详细需求、需求拆解、模块设计

## 产出文档
**路径**: `.opencode/docs/module-requirement-{module}.md`
**说明**: 模块级需求文档，包含模块职责、接口定义、交互流程

## 依赖文档
- `.opencode/docs/requirement-{feature}.md`（功能需求文档）

## 核心能力

1. **需求拆解**
   - 将功能需求拆解为模块
   - 定义模块边界和职责
   - 识别模块间依赖

2. **接口定义**
   - 输入/输出接口
   - 数据格式定义
   - 错误处理规范

3. **交互流程**
   - 用户交互流程
   - 模块间调用流程
   - 异常处理流程

## 输入要求

用户需提供：
- 功能需求文档
- 模块划分建议（可选）

## 输出格式

```markdown
# 模块需求：{模块名称}

## 模块职责

**所属功能**: {功能名称}
**核心职责**: {模块负责的功能}
**协作模块**: [相关模块列表]

- **验收标准**:
  - [ ] 标准 1
  - [ ] 标准 2

### FR-002: {功能点名称}
...

## 接口定义

### 输入接口
| 接口名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| userId | string | 是 | 用户 ID |
| options | object | 否 | 配置选项 |

### 输出接口
| 接口名 | 类型 | 说明 |
|--------|------|------|
| success | boolean | 是否成功 |
| data | object | 返回数据 |
| error | string | 错误信息 |

## 交互流程

```
用户 → [模块 A] → [模块 B] → 数据库
              ↓
           [错误处理]
```

### 正常流程
1. 用户触发操作
2. 模块 A 验证输入
3. 模块 B 处理业务逻辑
4. 返回结果

### 异常流程
1. 输入验证失败 → 返回错误提示
2. 业务处理失败 → 记录日志并返回错误

## 最小上下文原则

1. **聚焦当前模块** - 不深入其他模块实现
2. **接口清晰** - 明确定义输入输出
3. **便于实现** - 输出开发可直接使用的规格
