## 能力

### 1. 页面类型最佳实践

| 页面类型 | 核心要点 | 关键考量 |
|----------|----------|----------|
| **列表页** | 搜索/筛选/分页/排序 | 空状态、加载状态、批量操作 |
| **详情页** | 信息层级、操作入口 | 关键信息突出、辅助信息折叠 |
| **表单页** | 输入校验、提交反馈 | 实时校验、步骤引导、错误恢复 |
| **弹窗/抽屉** | 触发时机、内容聚焦 | 关闭方式、滚动穿透 |
| **操作反馈** | 成功/失败/警告 | 动画过渡、自动消失/手动关闭 |

### 2. 布局设计最佳实践

```markdown
## 列表页布局
- 搜索区域: 顶部固定，可折叠
- 工具栏: 操作按钮右对齐
- 表格: 斑马纹、行悬停、列对齐
- 分页: 底部固定，显示总数

## 最佳实践检查点
✓ 单页不超过 20 条关键信息
✓ 支持键盘导航
✓ 响应式适配移动端
✓ 固定表头，横向滚动
```

### 3. 交互设计最佳实践

| 场景 | 最佳实践 | 常见问题 |
|------|----------|----------|
| **点击反馈** | 按钮点击有态变化 | 无点击反馈 |
| **hover态** | 鼠标悬停提示详情 | 缺少hover |
| **加载中** | 显示骨架屏或loading | 空白等待 |
| **空状态** | 提供引导操作 | 只显示"暂无数据" |
| **错误提示** | 明确错误原因和解决 | 只显示"操作失败" |
| **表单校验** | 实时反馈，定位准确 | 提交后统一报错 |
| **确认操作** | 二次确认，危险操作警示 | 直接执行 |

### 4. 状态设计最佳实践

```markdown
## 必含状态
- 正常态: 完整功能展示
- 空状态: 引导用户创建内容
- 加载态: 骨架屏/loading
- 错误态: 明确错误信息+重试
- 成功态: 成功后跳转或提示
- 禁用态: 灰色+不可点击
- 部分态: 选中部分的状态

## 状态切换
- 加载→正常/错误
- 正常→编辑/详情
- 禁用→正常(满足条件)
```

### 5. 性能优化实践

| 优化点 | 实践方法 | 优先级 |
|--------|----------|--------|
| **首屏加载** | 骨架屏、懒加载 | P0 |
| **列表渲染** | 虚拟滚动、 pagination | P0 |
| **图片** | 懒加载、WebP、CDN | P1 |
| **接口** | 防抖、节流、缓存 | P1 |
| **交互** | 预加载、缓存结果 | P2 |

### 6. 无障碍设计 (a11y)

```markdown
## 必备无障碍特性
- 所有图片有 alt 描述
- 表单有 label 关联
- 按钮/链接可键盘聚焦
- 颜色对比度 ≥ 4.5:1
- 焦点可见 (focus ring)
- ARIA 标签正确使用

## 键盘导航
- Tab: 进入下一个焦点
- Enter: 触发操作
- Escape: 关闭弹窗
- 方向键: 导航菜单/列表
```
